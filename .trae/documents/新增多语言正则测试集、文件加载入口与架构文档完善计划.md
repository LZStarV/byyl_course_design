## 目录与样例文件
- 新增路径：`tests/regex/`
  - `tests/regex/python/identifiers.regex`（标识符、关键字、数字、字符串、空白、注释）
  - `tests/regex/python/operators.regex`（算术、比较、逻辑、赋值、分隔符）
  - `tests/regex/javascript/identifiers.regex`
  - `tests/regex/javascript/operators.regex`
- 规范：沿用现有宏与 Token 命名规范（`letter`/`digit`，`_ID101`、组 Token `_op200S` 等），每文件包含若干宏与 Token 定义，行内转义遵循现有规则。

## GUI 改造
- 菜单/工具栏
  - 菜单“文件→打开正则文件”（`QFileDialog`），读取文本到 `txtInputRegex`，状态栏提示并可自动或手动“转换”。
  - 菜单“示例→Python/JavaScript→(identifiers/operators)”：直接加载对应 `tests/regex/...` 文件到编辑器。
- 控件与 objectName
  - 新增按钮 `btnOpenRegexFile`（从文件加载），菜单对象名如 `menuLoadSamplePythonId`/`menuLoadSampleJsOp`。
- 逻辑
  - 文件打开失败使用 `QMessageBox`/`statusBar` 提示。
  - 路径拼接使用 `QDir/QFileInfo`，兼容跨平台；读取用 `QTextStream` 兼容 `\n/\r`。

## 引入流程与交互
- 手动输入：用户在 `txtInputRegex` 直接编辑 → 点击“转换” → 展示 NFA/DFA/MinDFA 表。
- 从文件选择：点击“打开正则文件” → 选中文本 → 填充编辑器 → 点击“转换”。
- 从示例菜单：直接填充编辑器 → 点击“转换”。
